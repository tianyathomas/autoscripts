# 拼多多果园自动化脚本

> 自动签到、浇水、领取任务奖励。支持青龙面板（QingLong）定时运行。

## 功能特性

| 功能 | 说明 |
|------|------|
| ✅ 自动签到 | 每日首次运行自动签到领取水滴 |
| ✅ 自动浇水 | 每次最多浇水 50 次（每 10 水滴/次） |
| ✅ 任务领取 | 自动接受并领取已完成的任务奖励 |
| ✅ Anti-Token | 自动下载拼多多风控 SDK，运行时动态生成，无需手动维护 |
| ✅ 任务探测 | 支持多种活动 ID（act=201015/201036）及 20+ 种任务类型 |

## 环境要求

- **Node.js** 14+
- **Python** 3.6+
- **requests** 库：`pip install requests`

## 快速开始

### 1. 获取 Cookie

1. 手机打开拼多多 → 果园页面
2. 打开浏览器开发者工具（F12）→ Network
3. 刷新果园页面，找任意请求，复制完整 Cookie 字符串
4. 脚本会自动从 Cookie 中提取 `pdd_user_id`，无需额外配置 UID

> [!IMPORTANT]
> 必须包含以下字段：
> - `api_uid`
> - `pdd_user_id`
> - `PDDAccessToken`
> - `pdd_user_uin`
> - `tubetoken`（重要，影响任务接口）
> - `pdd_vds`（重要，影响风控验证）

### 2. 本地运行

```powershell
# PowerShell 设置 Cookie
$env:PDD_COOKIE = 'your_cookie_string_here'

# 运行
node pdd.js
```

### 3. 青龙面板部署

#### 步骤一：添加环境变量

青龙面板 → 环境变量，添加：

| 名称 | 值 |
|------|-----|
| `PDD_COOKIE` | 完整 Cookie 字符串（必填，脚本自动从中提取 pdd_user_id）|

#### 步骤二：创建定时任务

青龙面板 → 定时任务，添加两条：

```
# 早上 8:00 - 签到 + 浇水
0 8 * * * node /ql/scripts/pdd/pdd.js

# 晚上 20:00 - 签到 + 浇水 + 领取任务
0 20 * * * node /ql/scripts/pdd/pdd.js
```

#### 步骤三：上传脚本

将整个 `pdd/` 文件夹上传到 `/ql/scripts/pdd/` 目录。

## 任务类型说明

### 可自动完成的任务 ✅

| 任务类型 | 说明 | 自动处理 |
|----------|------|----------|
| 签到 | 每日签到领水滴 | ✅ 自动签到 |
| 浇水 | 消耗水滴灌溉果树 | ✅ 每次 50 次 |
| 领取奖励 | 已完成任务的奖励 | ✅ 自动领取 |

### 需要真实浏览器行为 🔒

| 任务类型 | 说明 | 限制 |
|----------|------|------|
| 逛商品 | 浏览商品页面 N 次 | ❌ 需要真实浏览器访问 |
| 看视频 | 观看视频 N 秒 | ❌ 需要真实播放 |
| 逛店铺 | 访问店铺页面 | ❌ 需要真实浏览 |

> [!NOTE]
> 拼多多果园的任务分为两类：
> - **API 可完成**：签到、浇水、领取等，脚本完全自动化
> - **浏览类任务**：`finished_count` 需通过真实访问行为（如逛商品、看视频）才能累计，API 无法模拟，脚本只能负责**领取**已完成的奖励

## 工作原理

```
pdd.js (Node.js)
    │
    ├─[1] 下载拼多多风控 SDK（~100KB）
    │
    ├─[2] 在 Node.js 中模拟浏览器环境，执行 SDK
    │      生成 Anti-Token（绕过风控验证）
    │
    └─[3] 动态生成 Python 脚本 + 执行
           │
           ├─ 查询当前水滴数量
           ├─ 执行签到 API
           ├─ 执行浇水 API（带 Anti-Token）
           ├─ 获取任务列表（带 Anti-Token）
           ├─ 接受待接受任务
           └─ 领取可领取任务奖励
```

## 常见问题

**Q: 脚本运行失败，提示网络错误？**
A: 检查网络连接。脚本会自动设置 `NO_PROXY=*` 绕过代理，如仍有问题：
```powershell
$env:NO_PROXY = '*'
$env:no_proxy = '*'
node pdd.js
```

**Q: Anti-Token 生成失败？**
A: Anti-Token 依赖拼多多风控 SDK 的动态下载，若 CDN 不可达：
- 检查网络能否访问 `https://static.pddpic.com/`
- SDK URL 可能随拼多多版本更新变化，可搜索最新 URL 替换

**Q: Cookie 过期了怎么办？**
A: Cookie 通常 7~30 天过期。重新登录拼多多果园，复制新的 Cookie，更新青龙环境变量。

**Q: 为什么任务列表为空？**
A: 可能原因：
1. Anti-Token 生成失败 — 检查日志中 `[AntiToken]` 部分
2. Cookie 过期 — 重新获取 Cookie
3. 任务已全部领取 — 果园任务每日刷新，明天再看
4. API 接口变更 — 拼多多可能更新了接口，需抓包重新分析

**Q: 浏览类任务（done=0/9999）如何完成？**
A: 这类任务需要真实浏览器行为：
- 方案一：手动打开果园 App 完成浏览任务
- 方案二：使用 Playwright/Selenium 等浏览器自动化工具模拟访问
- 方案三：部分浏览任务支持在拼多多 App 内完成，脚本只负责领取奖励
